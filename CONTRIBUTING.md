# Contributing

Focused fixes that improve the UIZZE MCP documentation, agent skills, compatibility, or security are welcome.

## Before opening a pull request

1. Search existing issues and pull requests.
2. Keep the change scoped to one problem.
3. Verify every command or client configuration you change.
4. Do not commit UIZZE tokens, user data, private screenshots, generated credentials, or copied proprietary product content.
5. Preserve the free skill workflow and keep paid MCP capability described as an optional enhancement.
6. Link product calls to the plain homepage: https://uizze.com

For a substantial new integration, open an issue first with the target client, expected workflow, security model, and a minimal verification plan.
